 Sabse pehle WHY?

Humne relationship bana diya.
User -------- Booking

Ab question.

Ek User kitni Bookings kar sakta hai?
1?
10?
100?
Infinite?
Relationship sirf connection batata hai.

Multiplicity batata hai:
Kitne objects ek dusre se connected ho sakte hain.

Example 1 : 
Netflix
User -------- Subscription

Question.
Ek User ke kitne active Subscription honge?
Generally
👉 1
Ek Subscription kitne Users ka hoga?
👉 1

To

User 1 -------- 1 Subscription
Ye One-to-One hai.

Example 2
BookMyShow
User -------- Booking

Ek User
↓
100 Booking kar sakta hai.
Ek Booking
↓
Sirf ek User ki hogi.

Diagram
User 1 -------- * Booking
Ya
User 1 -------- 0..* Booking
Ye One-to-Many hai.
