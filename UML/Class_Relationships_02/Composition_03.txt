Composition
🤔 What is Composition?
Composition means one class strongly owns another class.
Composition = Has-A Relationship (Strong Ownership)
The child object's lifecycle completely depends on the parent object.

If the parent is destroyed, the child is also destroyed.

💡 Example 1
A House contains Rooms.
class House {
    private Room room;
    House() {
        room = new Room();
    }
}
UML
House ◆──── Room
✅ House creates Room.
✅ Room cannot exist independently.
❌ No other class should own this Room.

💡 Example 2
A Human has a Heart.
class Human {
    private Heart heart;
    Human() {
        heart = new Heart();
    }
}
Heart is created inside Human.
If Human dies, Heart also ceases to exist ➡️ Composition

💡 Example 3
A Car has an Engine.
class Car {
    private Engine engine;
    Car() {
        engine = new Engine();
    }
}
Engine belongs to this Car only. Destroy the Car → Engine is also destroyed ➡️ Composition

🎯 Easy Trick
Ask yourself: Who creates the object?
Parent creates it? ✅ Yes.
Then ask: Can the child exist without the parent? ❌ No.
➡️ Composition
