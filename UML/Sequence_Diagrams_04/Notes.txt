# 📘 Sequence Diagram

# 🤔 What is a Sequence Diagram?
A **Sequence Diagram** shows how different objects communicate with each other **over time** to complete a task.
It focuses on the **order of interactions** between objects.

> **Class Diagram** → What classes exist?  
> **Sequence Diagram** → How those classes interact at runtime?

---

# ❓ Why do we need it?
A Class Diagram only tells us the structure of the system.
A Sequence Diagram helps us understand:

- Which object starts the process?
- Which object calls the next object?
- What is the execution order?
- How the request and response flow through the system?

---

# 💡 Example: User Login

```text
👤 User
    |
    | Login Request
    v
LoginController
    |
    v
LoginService
    |
    v
UserRepository
    |
    v
Database
    |
    v
JWTService
    |
    v
Response
```

### Flow

1. User sends a login request.
2. Controller receives the request.
3. Service validates the user.
4. Repository fetches user data.
5. JWT token is generated.
6. Response is returned to the user.
