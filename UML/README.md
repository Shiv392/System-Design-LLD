# 📘 UML (Unified Modeling Language)

Welcome to the **UML** section of this repository.

This folder contains all the UML concepts required for **Low-Level Design (LLD)** interviews and real-world software development.

The goal is **not to memorize diagrams**, but to understand **how to model a software system before writing code.**

---

# 🎯 Why Learn UML?

Before writing code, every software engineer should be able to answer:

- What classes are required?
- How are those classes related?
- Which object owns another object?
- How many objects can be associated?
- Which object can access another object?

UML helps answer these questions visually.

---

# 📚 Prerequisites

Before learning UML, make sure you're comfortable with:

- ✅ OOP (Object-Oriented Programming)
- ✅ SOLID Principles

These concepts are heavily used while designing UML Class Diagrams.

# 💡 Interview Tip

Don't learn UML as a drawing tool.

Learn it as a **software design language**.

If you can explain **why** a relationship exists (instead of just drawing it), you're already thinking like a software engineer.
